### 2026-06-03 第二轮修复：页面刷新后全空白 Bug

**问题**: 数据加载成功（Data Checks 显示 56,573 条记录）但 KPI 全 0、表格/图表全空

**根因**: `buildMultiSelect()` 返回对象缺少 `_suppress()` 和 `getItems()` 方法。
`restoreFilterState()` 在恢复已保存筛选状态时调用 `msLane._suppress(true)` → TypeError → try/catch 捕获后返回 false。
而 `refreshAll()` 只存在于 try 块内部（第 1101 行），所以当 restore 失败时，**渲染从未执行**。
这同时解释了之前 Filter 持久化 Bug — 整个 `restoreFilterState` 在第一步就崩溃了。

**修复 5 处**:
1. `buildMultiSelect` 新增 `var _suppressed = false` 守卫变量
2. checkbox onchange / btnAll / btnNone 三个 `onChange(selected)` 调用处加 `if (!_suppressed)` 守卫
3. return 对象新增 `getItems()` 和 `_suppress(s)` 方法
4. 数据加载后无条件调用 `refreshAll()`（`restoreFilterState()` 成功/失败都渲染）

**部署**: index.html + db_data.json 均已推送 GitHub (sha aac2668 / 1dacc12)
